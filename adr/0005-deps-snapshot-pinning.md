# ADR 0005 — Dependency snapshot pinning via `_deps.snap`

**Status:** Accepted

## Context

The earliest version of dependency tracking used `deps.test.ts` files that
snapshotted resolved dependencies and failed CI when versions drifted. The
format was test-file-shaped and noisy. With many workspaces and frequent JSR
and `@std` bumps, the test-based approach made diffs hard to review and
mixed dependency drift with real test failures.

## Decision

Each package has per-entrypoint `_deps.snap` files generated by
`_tools/check_deps.ts`. The lint task `lint:deps` checks that current
resolution matches the snapshot. Snapshots are updated by running:

```bash
deno run -A _tools/check_deps.ts --update
```

## Consequences

- Version bumps in `import_map.json` always require updating snapshots in
  the same commit.
- Snapshot diffs are JSON-shaped and reviewable.
- New workspaces must include `_deps.snap` files alongside their entrypoints.
- Dependency drift is separated from real test failures.
